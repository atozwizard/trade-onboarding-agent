# RiskManaging Agent Presentation Content

## 1. 확장성 아이디어 (Scalability Ideas)
- **실시간 외부 리스크 API 연동**: 글로벌 물류 뉴스(파업, 기상, 재해) 및 선박 추적 API를 연동하여 외부 리스크를 실시간으로 자동 감지 및 경고.
- **Enterprise ERP 연동**: 사내 계약 관리 시스템과 연동하여 계약 금액, 패널치 조항 등 컨텍스트를 수동 입력 없이 자동 로드하여 분석 정밀도 향상.

## 2. 기능별 발표 포인트 (Key Features)
- **협업형 대화 가이드**: 사용자가 정보를 모두 주지 않아도 대화를 통해 필요한 데이터를 스스로 수집하는 '상호작용형 분석'.
- **지식 공유 및 교육**: 리스크 산출 기준과 로직을 전문적으로 설명해주는 '선배/상사 페르소나'를 통한 실무 가이드 제공.
- **RAG 기반 근거 제시**: 과거 유사 클레임 사례 및 사내 매뉴얼(RAG)에 기반한 객관적인 대응 전략 수립.

## 3. 시연 시나리오 (Demo Scenarios)

### STEP 1: 리스크 징후 감지 및 대화 시작
- **Query**: "수출 건 선적 지연이 5일 발생할 것 같아. 리스크가 클까?"
- **Point**: 에이전트가 리스크 상황을 인지하고, 분석에 필요한 필수 정보(계약금, 패널티 등)를 역으로 질문함.

### STEP 2: 전문적 기준 설명 및 데이터 수집
- **Query**: "계약금은 5만 달러야. 근데 리스크 점수는 어떻게 산정하는 거야?"
- **Point**: 시스템 내 정의된 영향도(Impact)와 발생 가능성(Likelihood)의 산출 방식을 설명하며 신뢰 형성.

### STEP 3: 최종 보고서 승인 및 생성
- **Query**: "분석 결과를 보고서로 정리해줘."
- **Point**: 사용자의 명시적 승인 후, 정량적 점수 + 유사 사례 + 대응 전략이 포함된 구조화된 리스크 레포트 출력.

## 4. 발표 스크립트 (Draft)
저희 리스크 관리 에이전트는 단순한 정보 조회를 넘어, 사용자와 함께 위험을 진단하는 '협업형 리스크 관리'를 지향합니다.
사용자가 "지연 발생"과 같은 고민을 털어놓으면, 에이전트는 즉시 상황을 파악하고 선배처럼 필요한 정보들을 묻기 시작합니다. 
이 과정에서 사용자는 리스크의 기준이 무엇인지, 현재 상황이 얼마나 위험한지 에이전트와 대화하며 실무 지식을 학습하게 됩니다.
최종적으로는 RAG를 통해 조회된 실제 유사 사례와 전문적인 스코어링을 결합하여, 실무에서 즉시 보고용으로 사용할 수 있는 대응 전략 보고서를 생성하며 마무리됩니다.
