생성된 퀴즈를 원본 RAG 검색 데이터와 대조하여 정확성을 판단합니다.

## 검증 항목
각 문제에 대해 아래 5가지를 확인하세요:

1. **문제(question)**: 원본 데이터의 용어 또는 설명을 올바르게 반영했는가?
2. **정답 선택지**: choices[answer] 가 원본 데이터와 일치하는가?
3. **오답 선택지**: 나머지 3개 선택지가 실제 존재하는 다른 무역 용어나 설명 기반인가? (완전한 허구가 아닌가?)
4. **정답 인덱스(answer)**: answer 인덱스가 실제 정답을 가리키는가? (0=첫 번째, 1=두 번째, ...)
5. **해설(explanation)**: 해설 내용이 원본 데이터와 부합하는가?

## 판정 기준
- **is_valid = true**: 위 5개 항목 모두 원본 데이터와 부합
- **is_valid = false**: 하나라도 원본 데이터와 불일치하거나 허구 포함

## 출력 형식
반드시 아래 JSON 배열 형식으로만 응답하세요. JSON 외의 텍스트는 절대 포함하지 마세요.

```json
[
  {
    "quiz_id": "문제의 quiz_id",
    "is_valid": true,
    "issues": []
  },
  {
    "quiz_id": "문제의 quiz_id",
    "is_valid": false,
    "issues": ["구체적인 불일치 사유"]
  }
]
```

- issues 배열에는 불일치하는 항목과 구체적 사유를 명시하세요.
- is_valid가 true이면 issues는 빈 배열 [] 이어야 합니다.
